# Quail 音频设计调研与方案 (Audio Design Research & Plan)

## 1. Kahoot 声音设计调研 (Research)

Kahoot 的音频设计遵循 **“压力与释放 (Tension & Release)”** 的心理学原则：

*   **等待大厅 (Lobby)**：
    *   **设计感**：节奏轻快、循环性强、带有期待感。
    *   **作用**：建立“游戏即将开始”的心理预期，缓解等待的枯燥。
*   **答题倒计时 (The "Thinking" Music)**：
    *   **设计感**：这是最经典的部分。音乐从简单的低音开始，随着时间流逝，层级不断丰富，节奏加快，音调升高。
    *   **作用**：制造**紧迫感**。它迫使玩家在最后几秒产生“必须快点决定”的冲动，增加竞技压力。
*   **反馈音效 (Feedback SFX)**：
    *   **正确**：高频、清脆、上升音阶（类似金币收集）。
    *   **错误**：低频、沉闷、下降音阶（类似“呜呼”声）。
    *   **作用**：提供即时的多维度反馈，强化成功的喜悦或失败的遗憾。
*   **排行榜与领奖台 (Leaderboard & Podium)**：
    *   **设计感**：戏剧性、宏大、充满胜利感。
    *   **作用**：给予高分玩家仪式感，激励其他玩家。

## 2. Quail 音频增强方案 (Proposed Plan)

### A. 音频场景规划
| 场景 | 声音类型 | 描述 |
| :--- | :--- | :--- |
| **Lobby** | 背景音乐 (BGM) | 轻松、律动感强的 Lo-fi 或 8-bit 音乐。 |
| **Lobby** | 音效 (SFX) | 玩家加入时播放清脆的 "Pop" 或 "Ding" 声。 |
| **Question** | 背景音乐 (BGM) | 紧张、有节奏的鼓点，最后 5 秒自动加速/升调。 |
| **Result** | 音效 (SFX) | 答对：清脆的成功铃声；答错：短促的失败提示。 |
| **Leaderboard** | 背景音乐 (BGM) | 充满悬念的鼓点，等待排名滚动。 |
| **Podium** | 背景音乐 (BGM) | 欢快、宏大的胜利乐章。 |

### B. 技术实现策略
1.  **全局音频管理器**：使用 Web Audio API 封装一个 `AudioManager`，支持平滑淡入淡出。
2.  **浏览器限制处理**：在首页增加“开启声音”交互，激活音频上下文。
3.  **音量控制**：主持人端全局调节，玩家端静音开关。
4.  **资源选择**：使用免版税 (Royalty-free) 的游戏音频素材。
