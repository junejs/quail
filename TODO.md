# Quail Project TODO List

This document outlines the planned improvements and future features for the Quail quiz application.

## 1. 游戏机制与互动增强 (Game Mechanics)
- [ ] **双倍积分/道具系统**：引入 "Power-ups"（如连续答对 3 题获得双倍积分机会）。
- [ ] **速度奖励可视化**：在玩家端实时显示预计得分，强化“越快得分越高”的反馈。
- [ ] **玩家互动**：允许玩家在等待时发送“表情雨”或互动信号，实时浮现在主持人屏幕上。

## 2. 内容管理与持久化 (Content & Persistence)
- [ ] **本地存储持久化**：使用 Zustand 的 `persist` 中间件将测验数据保存到 `localStorage`。
- [ ] **测验管理功能**：增加对已有测验的 **编辑** 和 **删除** 功能。
- [ ] **导入/导出功能**：支持 JSON 格式分享测验，或从 Excel/CSV 批量导入题目。

## 3. 视觉与动效优化 (UI/UX & Motion)
- [ ] **响应式布局深度优化**：
    - [ ] **主持人端**：优化超大屏幕（4K）下的布局，增加背景装饰动效。
    - [ ] **玩家端**：优化横屏模式下的操作体验。
- [ ] **转场动画增强**：在题目展示、答题结果、排行榜之间增加更具冲击力的 3D 或缩放转场。
- [ ] **色盲友好模式**：提供高对比度或特定色盲模式的配色方案。

## 4. 主持人控制力 (Host Controls)
- [ ] **实时状态监控**：主持人端显示具体哪些玩家尚未答题（头像亮灭状态）。
- [ ] **踢人功能**：在 Lobby 阶段允许主持人移除不当昵称的玩家。
- [ ] **游戏控制**：增加暂停计时或跳过当前题目的功能。

## 5. 技术健壮性 (Technical Robustness)
- [ ] **断线重连逻辑**：实现基于 `localStorage` 的 Session 恢复，防止刷新页面丢失进度。
- [ ] **服务器端验证**：将核心得分逻辑完全移至服务器端，防止客户端篡改。
- [ ] **单元测试与集成测试**：为核心 Socket 逻辑和状态管理编写测试。
